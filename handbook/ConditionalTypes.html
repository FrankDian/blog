    # 运行机制

    # 作用域

    JS采用的是词法作用域(lexical scoping)，也叫静态作用域，即在定义的时候就把函数的作用域确定了，相对应的是动态作用域，要在运行的时候才能确认。 以如下代码为例：

    var value = 1;
function foo() {
    console.log(value);
}
function bar() {
    var value = 2;
    foo();
}
bar();

    词法作用域的输出结果为1，因为运行foo函数的时候，看的是定义时候的作用域。 动态作用域的输出结果为2，是在运行位置找上一层的作用域，即为bar函数的作用域。 实际使用中，Bash语言使用的是动态作用域。

    # 作用域链

    以下面一段函数为例：

    var a = 1; // 全局作用域加入此变量
function test () {
  var b = 2;
}
test();

    在执行到第二行的时候，一个标识符（identifier）被加入当前作用域对象（全局作用域对象），并且指向一个函数对象（function object），此对象保存着函数的源码及其他属性，最重要的就是有个**[[scope]]**属性，指向的是当前的作用域对象。 简而言之就是通过[[scope]]对象来保存对定义时的父作用域的引用，形成了作用域链，当函数被调用的时候，如果某个函数在当前作用域中找不到，会通过 [[scope]]属性去定义时的父作用域中查找，这与原型链非常相似，原型链如果找不到的话会返回 undefined，作用域链中如果找不到则会返回 Reference Error。

    # 执行上下文 - execution context

    执行上下文就是指在执行的时候能够有权访问的数据和函数，还有要执行的具体代码。 JS代码中类型有三种：

    • 全局上下文
    • 函数上下文
    • eval函数上下文（略）

    在代码执行的时候，会有一个执行上下文栈来管理上下文，保存的就是当前的执行上下文，比如执行到了一个函数，就会创建一个函数上下文，然后入栈，执行完了之后再出栈。 创建执行上下文步骤：

    • this绑定：确定this的指向。
    • 创建词法环境：用来存储 函数定义、let、const定义
    • 创建变量环境：用来存储 var变量定义，在初期会置为undifined，所以有变量提升。

    # 执行上下文栈

    JavaScript 引擎用执行上下文栈（Execution context stack，ECS）来管理执行上下文，ECStack就是一个栈的结构，里面保存的就是当前的执行上下文，比如执行到了一个函数，就会创建一个函数上下文，然后入栈，执行完了之后再出栈； 首先都会先创建一个全局上下文，比如如下的例子：

    function fun2() {
    console.log('执行完毕');
}
function fun1() {
    fun2();
}
fun1();
// xxx其他代码

    const ECStack = [];
ECStack.push(GlobalContext); // 全局上下文对象
ECStack.push(func1Context); // func1 执行上下文对象
ECStack.push(func2Context); // func2 执行上下文对象
ECStack.pop(); // func2 执行完毕
ECStack.pop(); // func1 执行完毕
// javascript接着执行下面的代码，但是ECStack底层永远有个globalContext

    练习：下面两段代码执行上下文有什么区别？ 两段代码的输出结果一样，但是执行上下文栈的变化不一样。

function checkscope(){
    var scope = "local scope";
    function f(){
        return scope;
    }
    return f();
}
checkscope();

// 代码二

    function f(){
        return scope;
    }

    # 闭包

    返回的内部函数被未被释放，而内部函数中引用了外部函数的变量，所以根据作用域链的原理，内部函数的**[[scope]]**属性一直引用着定义时的父作用域对象，即外部函数的作用域对象，导致其不能被垃圾回收；并且这个引用的外部函数作用域对象只能通过返回的内部函数标识符，形成类似于私有变量的状况。 以代码为例：

    "use strict";

var elems = document.getElementsByClassName("myList");
for (i = 0; i < elems.length; i++) {
  elems[i].addEventListener("click", function () {
    this.innerHTML = i;
  });
}

    在循环中创建了很多的函数作用域，每个作用域的 [[scope]]属性都引用了同一个父作用域中的 i 变量，所以最终输出的都是同样的值。

    # 闭包的作用

    封装变量，收敛权限

    // 实现防抖debounce: 过一段时间才会触发事件，事件会被刷新推迟
// 场景：搜索框的输入推荐。
function debounce(fn, time) {
  let timer;
  return function() {
    clearTimeout(timer);
    timer = setTimeout(() => {
      fn();
    }, time);
  }
}

// 实现节流 throttle：触发完事件之后，一段时间内都不触发。
// 场景：点击获取页面最新数据。
function throttle(fn, time) {
  let flag = false;
  return function() {
    if(flag) return;
    flag = true;
    fn();
    setTimeout(()=>{
      flag = false;
    }, time);
  }
}

    # prototype

    原型（prototype）承担的职能是给其他对象提供共享属性访问的对象，只有函数有prototype。 在标准发布时prototype是一个隐式对象，通过Object.getPrototypeOf() / Object.setPrototypeOf() 方法来分别进行获取和设置；

    # proto

    但是由于已经有部分浏览器通过__proto__来访问到prototype对象，所以只好纳入规范。 所有的引用类型都有__proto__这个属性。 实际上__proto__这个属性不是真实存在的，用for in 和Object.keys() 均无法遍历出来。实际是通过给Object.prototype 设置 __proto__对象的get/set 来实现的，类似如下：

    const test = {
  get a() {
    console.log('get');
    return Object.getPrototypeOf(this);
  },
  set a(value) {
    console.log('set');
    return Object.setPrototypeOf(this, value);
  }
};
test.a === test.__proto__  // true

    # constructor

    此属性为对象所特有的，指向一个函数，即为该对象的构造函数。

    const b = function(){};
b.prototype.constructor === b // true
const c = new b();
c.__proto__ === b.prototype // true

    # 原型链

    每个对象都有一个隐式对象指向其原型，而原型也是对象，也有隐式对象，所以就构成了原型链。 总结：

    1. 所有的引用类型都有一个__proto__属性，属性值是一个普通对象。
    2. 所有的函数都有一个prototype属性，值也是一个普通对象。
    3. 所有引用类型值的__proto__都指向其构造函数的prototype。
    4. Object.prototype.proto 是null => 为了防止死循环

    # 原型链有什么作用？

    在查找一个对象的属性的时候，如果在当前找不到的话，会通过原型链去下一个节点上找。

    # 原型链查找代码模拟

    const lookupProperty = (object, propertyName) => {
  let current = object;
  if(current === null) {
    throw new Error(`Can't read property ${propertyName} of ${object}`);
  }
  while(current) {
    // 防止原型上有{hasOwnproperty: () => true }这种属性
    if(Object.prototype.hasOwnProperty.call(current, propertyName)) {
      return current[propertyName];
    }
    current = Object.getPrototypeOf(current);
  }
  return undefined;
}
lookupProperty({}, 'toString') === Object.prototype.toString // true

    # 单线程和异步

    单线程就是两段JS代码不能同时运行，目的是避免 DOM 渲染冲突。 所以 JS 代码在执行的时候浏览器 DOM 渲染是停止的（JS引擎线程 和 GUI渲染线程是互斥的）。 Web Worker 可以做一些耗时的 JS 计算，但是不能操作 DOM。 JS 的异步是通过事件轮询机制来完成的。

    # 事件轮询(Event Loop)

    Event Loop是在HTML Standard中定义的，不是在ECMAScript定义，是JS实现异步的机制，具体如下：

    • 异步任务进入EventTable并注册函数，等待任务条件满足的时候，放入Event Queue
    • 同步任务在主线程执行
    • 在Call Stack空闲的时候，触发 Event Loop 机制
    • Monitoring Process 会在主线程空闲的时候，轮询Event Queue，执行异步任务
    • 异步任务的执行顺序是先将MicroTask清空，然后再执行MacroTask
      • 顺序：同步代码 -> 清空MicroTask Queue -> 一个宏任务 -> 再清空MicroTask Queue;
      • 执行顺序的循环是：执行一个宏任务，清空微任务队列
    • 在清空MicroTask Queue之后，浏览器有可能会更新视图，一帧内的多次DOM变动是不会立即响应的。
    • MacroTask: JS的同步代码 / setTimeout() / setInterval() / IO / UI Rendering
    • MicroTask: Promise(ECMAScript规范，有些浏览器放入MacroTask) / process.nextTick() / MutationObserver
    • 浏览器可以有多个Event Loop，比如 Browsing context（如一个标签页） 和 Web worker都有自己的Event Loop。
    • 所有同源的 Browsing Contexts可以共用一个Event Loop，以便他们之间的相互通信。

    # Node中与浏览器Event Loop的区别

    浏览器中的Event Loop是在每个宏任务之后清空微任务队列。 而Node的Event Loot是在libuv库中实现，分为6个阶段，每个阶段之后都清空微任务队列，6个阶段分别为：Timers、IO callbacks、idles prepare、poll、check、close Callbacks 总结： 浏览器端：微任务队列在宏任务执行完毕之后执行。 Node端：微任务队列在事件循环的各个阶段之间执行。

    # Service Worker

    浏览器后台运行的脚本，独立于Web 页面，有以下特性：

    • 必须运行于HTTPS或localhost协议上：为了安全和本地调试方便。
    • 不能使用window和操作DOM，防止影响页面渲染，造成冲突。
    • 生命周期有三个：下载，安装，激活。
    • 非常强大，类似于浏览器和服务器之间的一个代理服务器，所以可以用来接收推送和做离线缓存。
    • 可以集中接收数据，比如地理数据和陀螺仪数据，然后被多个页面共享。

    # PWA

    Progressive Web App，致力于将Web App打造成跟原生手机App一样的体验。 特点：

    1. 创建APP图标，基于Manifest文件。
    2. 离线应用：基于Service Worker，拦截网络请求，判断当前是否离线，然后决定是采用缓存数据，还是更新缓存数据等操作。
    3. 消息推送。

    # WebWorker

    • HTML5规范中定义：脚本可以运行在浏览器后台的线程中，这是浏览器提供的一个能力；
    • 种类：Dedicated Web Worker（默认Worker）只能一个页面使用，SharedWorker可以多个页面共用，遵守同源策略；
    • 不能执行操作DOM的JS,防止与主线程的操作冲突；
    • 有自己独立的执行上下文，所以不能访问到window；
    • 通过onmessage和postmessage来跟主线程交互；
    • 传递的是数据的拷贝，不是数据的引用，因为在传递的时候会在两端序列化/反序列化，类似于JSON.stringify() 和 JSON.parse()；
    • 常用方法：importScripts() / terminate() / close()
    • 常用场景：
      • 大量的计算可以放入到web worker中，比如在线编辑图片中涉及到的大量计算。
      • 大量的请求或者轮询任务；
      • 音视频的解析任务；

    # 深浅拷贝

    对于引用对象来说，如果只是通过等于号，那就是浅拷贝，获得的只是一个引用。 深拷贝就是完全的拷贝，新对象和老对象内容一样但是不会相互影响。 利用递归和for in循环来将数组和对象一一深复制出来，代码实现如下：

    function deepClone(source) {
  const targetObj = source.constructor === Array ? [] : {};
  for(let key in source) {
    if(!source.hasOwnProperty(key)) continue;
    if(source[key] && typeof source[key] === 'object') {
      targetObj[key] = deepClone(source[key]);
    } else {
      targetObj[key] = source[key];
    }
  }
  return targetObj;
}

// Test
const originObj = {
  a:'a',
  c:[1,2,3],
  d:{dd:'dd'},
  sayHello: function(){
    console.log('Hello World');
  }
};
const cloneObj = deepClone(originObj);
console.log(cloneObj === originObj); // false
cloneObj.a = 'aa';
cloneObj.c.push(4);
cloneObj.d.dd = 'doubled';
console.log(cloneObj); // {a:'aa',c:[1,2,3,4],d:{dd:'doubled'}, sayHello: xxx};
console.log(originObj); // {a:'a',c:[1,2,3],d:{dd:'dd'}, sayHello: xxx};

    # 作用域和作用域链

    ES6之前没有块级作用域，只有函数和全局作用域，用一个立即执行函数可以模拟块级作用域。 ES6有let和const有块级作用域。 当一个变量在当前作用域没有的话，会在定义时的父级作用域中找，找不到的话在更上一级的父级作用域找，这就是形成的作用域链。

    # 同步和异步的区别是什么

    同步会阻塞线程，异步不会； alert是同步，setTimeout是异步，异步有微任务和宏任务。 JS是单线程，但是浏览器不是，只是执行JS代码的引擎是个单线程的所以JS的代码没办法开启多个线程，但是浏览器还有定时器线程、事件触发线程、异步http请求线程、GUI线程。 所以在执行定时器、事件、ajax这些异步事件的时候是另外三个线程在执行代码，并不是JS引擎在做事情，在这些线程达到某一特定事件把任务放入JS引擎的线程中，同时GUI线程（渲染界面HTMl的线程）与JS线程是互斥的，在JS引擎执行时GUI线程会被冻结、挂起。

    # script 标签 defer 和 async 的区别

    普通 script 标签：会阻塞下载，然后执行之后继续渲染下面的内容。 defer 的 script 标签：并行下载，等待解析完 HTML 文件之后再执行。 sync 的 script 标签：并行下载，下载完了直接阻塞执行。

    • 相同点
    1. 只用于外联script，<script>没有src属性时忽略该defer、async属性；
    2. script下载与HTML解析并行。
    • 不同点
    1. defer script的执行在HTML解析完成之后，而async script的执行会阻塞HTML解析；
    2. 多个defer script的执行顺序与它们在document出现的顺序一致，async script的执行顺序依赖于它们下载完成的快慢。
    3. 同时使用defer、async，async的优先级更高。

    应用：一般有依赖关系的比如 jQeury 这种用 defer，像数据埋点这种用 async。

    # 代码模拟new操作符

    分为以下三步：

    1. 以构造函数的prototype为原型，创建新对象
    2. 将this（上一步中创建的新对象）和参数传递给构造函数，执行

  // 1
  const newObj = Object.create(Func.prototype);
  // 2
  const result = Func.apply(newObj, rest);
  // 3
  return typeof result === 'object' ? result : newObj;
}

    # 数组扁平化转换

    在说到模版编译的时候，有可能会提到数组的转换，一般就用递归处理 将 [1,2,3,[4,5]] 转换成

    {
    children:[
        {
            value:1
        },
        {
            value:2
        },
        {
            value:3
        },
        {
            children:[
                {
                    value:4
                },
                {
                    value:5
                }
            ]
        },
    ]
}

    // 测试数组
var arr =[1,2,3,[4,5]];
// 转换函数
function convert(arr){
    //准备一个接收结果数组
    var result = [];
    // 遍历传入的 arr 的每一项
    for(let i=0;i<arr.length;i++){
        //如果遍历到的数字是number，直接放进入
        if(typeof arr[i] == 'number'){
            result.push({
                value:arr[i]
            });
        } else if(Array.isArray(arr[i])){
            //如果遍历到这个项目是数组，那么就递归
            result.push({
                children: convert(arr[i])
            });
        }
    }
    return result;
}

var o = convert(arr);
console.log(o);
